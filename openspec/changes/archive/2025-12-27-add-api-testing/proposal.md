# 变更：添加 API 测试功能

## 为什么
服务端的所有接口目前缺乏系统化的测试覆盖。为了确保 API 的可靠性和正确性，需要：
1. 为每个接口或关键流程编写单元测试或接口测试
2. 提供统一的测试入口和结果输出
3. 提供可视化的测试界面，方便开发者和测试人员验证接口功能
4. 为后续 AI 接口测试提供扩展基础

## 变更内容
- 为所有 API 接口创建测试用例（单元测试或接口测试）
- 创建测试数据管理机制（使用独立的测试作品和剧集）
- 创建命令行测试脚本入口，支持批量执行和结果输出
- 创建前端测试页面，提供可视化的测试用例管理界面
- 支持单个用例执行、批量执行和结果详情查看

## 影响
- **影响的规范**：无（新增测试能力）
- **影响的代码**：
  - `server/` - 新增测试模块
  - `apps/web/` - 新增测试页面
- **数据存储**：测试数据存储在独立的测试数据目录中，不影响生产数据
